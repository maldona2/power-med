# Frontend - React

- Use functional components with hooks (no class components).
- Keep components small and focused on a single responsibility.
- Extract reusable logic into custom hooks.
- Use TypeScript for all components and hooks.
- Props should be typed with interfaces or types.
- Avoid prop drilling; use Context API for deeply nested state.
- Use `React.memo()` for expensive components that re-render frequently.
- Handle loading and error states in data-fetching components.
- Clean up effects with return functions to prevent memory leaks.
